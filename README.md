## ⚡ Quick Reference — Complexity Cheat Sheet

```
DATA STRUCTURE OPERATIONS:
Array:         Access O(1)  | Search O(n)  | Insert O(n)  | Delete O(n)
Linked List:   Access O(n)  | Search O(n)  | Insert O(1)* | Delete O(1)*
Stack:         Push O(1)    | Pop O(1)     | Peek O(1)
Queue:         Enqueue O(1) | Dequeue O(1) | Peek O(1)
Hash Table:    Search O(1)* | Insert O(1)* | Delete O(1)*  (* average)
BST:           Search O(h)  | Insert O(h)  | Delete O(h)   (h = height)
Balanced BST:  Search O(logn)| Insert O(logn)| Delete O(logn)
Heap:          Insert O(logn)| Extract O(logn)| Peek O(1)
Graph BFS/DFS: O(V + E)     (V = vertices, E = edges)

SORTING ALGORITHMS:
Bubble Sort:    O(n²)  time | O(1)    space
Selection Sort: O(n²)  time | O(1)    space
Insertion Sort: O(n²)  time | O(1)    space | Best: O(n)
Merge Sort:     O(nlogn) time | O(n)  space | Stable ✓
Quick Sort:     O(nlogn) avg | O(logn) space | Worst O(n²)
Heap Sort:      O(nlogn) time | O(1)  space
Counting Sort:  O(n+k)  time | O(k)  space | (k = range)

SEARCHING:
Linear Search:  O(n)
Binary Search:  O(logn)  — requires sorted array

BIG O ORDER (fastest to slowest):
O(1) < O(logn) < O(n) < O(nlogn) < O(n²) < O(2ⁿ) < O(n!)
```
